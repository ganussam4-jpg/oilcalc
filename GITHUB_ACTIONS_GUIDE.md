# Сборка APK через GitHub Actions с телефона

Полная инструкция: от регистрации до скачивания готового APK.
**Делается один раз. Компьютер не нужен.** Нужен только Android-телефон с интернетом
(на базе, не в поле).

---

## Что в итоге получится

После первого push'а в GitHub:
- В облаке Ubuntu автоматически собирается APK (~30-45 мин в первый раз).
- Файл `oilcalc-1.0.0-...-debug.apk` доступен для скачивания во вкладке Actions.
- Скачиваете APK на телефон, копируете на любые другие телефоны без интернета.

При каждом изменении (например, поправили формулу) — пересборка автоматически за 5-10 минут.

---

## Шаг 1. Регистрация в GitHub (если ещё нет аккаунта)

1. Открыть в браузере телефона: **https://github.com/signup**
2. Ввести email, пароль, придумать username (например, `ivanov-oil`).
3. Подтвердить email (на почту придёт письмо с кодом).

GitHub бесплатно даёт **2000 минут сборки в месяц** на приватных репозиториях и
**безлимитно** на публичных. Нам хватит с большим запасом.

---

## Шаг 2. Создать репозиторий

1. На главной странице GitHub нажать **зелёную кнопку «New»** (или плюсик вверху → «New repository»).
2. Заполнить:
   - **Repository name**: `oilcalc` (любое имя)
   - **Public** (бесплатная безлимитная сборка) или **Private** (если не хотите, чтобы код видели; 2000 мин/мес тоже хватит)
   - Поставить галочку **«Add a README file»** (потом перезапишем).
3. Нажать **Create repository**.

---

## Шаг 3. Загрузить файлы с телефона

Самый простой способ — через веб-интерфейс GitHub:

### 3.1. Скачать архив

1. Скачать `oilcalc_apk.zip` на телефон в папку Download.
2. Распаковать архив любым файловым менеджером (например, **Files by Google**, **ZArchiver**, **Cx File Explorer**).
   Должна получиться папка `oilcalc_apk` с файлами:
   ```
   main.py
   calc_engine.py
   buildozer.spec
   README.md
   .gitignore
   .github/workflows/build-apk.yml
   ```

### 3.2. Залить файлы в репозиторий

В открытом репозитории на GitHub:

1. Нажать **Add file → Upload files**.
2. Перетащить (или выбрать через «choose your files») **все файлы** из распакованной папки — **кроме папки `.github`**, её зальём отдельно.
3. Внизу — кнопка **Commit changes**.

### 3.3. Залить workflow-файл (особый случай)

Папка `.github/workflows/` — это специальная папка GitHub, её надо создать вручную:

1. На главной странице репозитория нажать **Add file → Create new file**.
2. В поле имени файла ввести **именно так** (со слешами):
   ```
   .github/workflows/build-apk.yml
   ```
   (как только введёте слеш, GitHub автоматически превратит это в папку)
3. В большое поле снизу скопировать содержимое файла `build-apk.yml` из архива
   (открыть его файловым менеджером в текстовом редакторе и скопировать целиком).
4. Внизу — **Commit changes → Commit directly to main → Commit changes**.

---

## Шаг 4. Запустить сборку

Как только workflow-файл закоммичен, сборка начинается **автоматически**.

1. Перейти во вкладку **Actions** в репозитории (вверху).
2. Увидите запуск с именем коммита (например, «Add build-apk.yml»).
3. Жёлтый кружок = идёт сборка. Зелёная галочка = готово. Красный крест = ошибка.

Первая сборка идёт **30-45 минут** (скачивается Android SDK, NDK, компилируется Python, Kivy).
Можно закрыть телефон и заняться делами — GitHub соберёт сам.

---

## Шаг 5. Скачать APK

Когда сборка завершилась с зелёной галочкой:

1. Открыть завершённый run во вкладке **Actions**.
2. Прокрутить вниз до раздела **Artifacts**.
3. Нажать на **oilcalc-apk** — скачается ZIP-файл с APK внутри (это особенность GitHub, APK всегда в ZIP-обёртке).
4. Открыть скачанный ZIP файловым менеджером, достать оттуда `.apk`-файл.

---

## Шаг 6. Установить APK на телефон

1. Открыть APK файловым менеджером.
2. Android спросит разрешение «Установка из неизвестных источников» — **Разрешить**.
3. Нажать **Установить**.
4. Иконка **OILCALC** появится в меню приложений.

Запустить — приложение работает офлайн.

---

## Перенос на другие телефоны (без интернета)

Один раз скачанный APK можно ставить на любое количество телефонов:
- Передать через Bluetooth.
- Скопировать на флешку и через OTG-кабель.
- Залить через ПК.

На месторождении интернет не нужен — APK самодостаточный.

---

## Внести изменение и пересобрать

Если потребуется поправить формулу или добавить расчёт:

1. В репозитории зайти в нужный файл (например, `calc_engine.py`).
2. Нажать карандашик «Edit this file» вверху справа.
3. Внести правки, внизу **Commit changes**.
4. Сборка запустится автоматически. Через ~5-10 минут (кэш ускоряет) — новый APK во вкладке Actions.

---

## Если сборка упала с ошибкой (красный крест)

1. Открыть упавший run во вкладке Actions.
2. Кликнуть на джоб **Build APK for Android**.
3. Развернуть шаг, который упал (там тоже будет красный крест).
4. Скопировать последние 30-50 строк лога и прислать мне — разберём.

Типичные причины:
- В первый раз скачка SDK/NDK тайм-аутится на медленной сети GitHub — просто перезапустить
  (Actions → нужный run → «Re-run all jobs»).
- Изменили `buildozer.spec` некорректно — откатить.

---

## Ограничения GitHub Actions

- **Бесплатно**: 2000 минут/мес для приватных репо, безлимит для публичных.
- **Время одной сборки**: до 6 часов (нам нужно меньше часа).
- **Размер артефакта**: до 500 МБ (наш APK ~30-50 МБ).

Для нашего проекта всё с большим запасом.
